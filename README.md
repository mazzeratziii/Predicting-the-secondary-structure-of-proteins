# Предсказание вторичной структуры белков

## 1. Описание проекта
Данный проект посвящен решению задачи биоинформатики: предсказание доминирующей вторичной структуры белка на основе его аминокислотной последовательности. Вторичная структура кодируется тремя состояниями: спираль (H), лист (E) и петля (C). В работе сравниваются различные методы представления последовательностей и алгоритмы машинного обучения.

## 2. Набор данных
*   **Датасет:** Protein_dataset https://www.kaggle.com/datasets/tranminhthuan/protein-dataset
*   **Источник:** Файл `train.csv`.
*   **Размер датасета:** 5534 образца после очистки.
*   **Структура:** Содержит колонки `seq` (аминокислотная последовательность), `sst8` (последовательность вторичной структуры в 8-х классовом состоянии) и `sst3` (последовательность вторичной структуры в 3-х классовом состоянии).
*   **Целевая переменная (`target`):** Доминирующий класс вторичной структуры для всего белка, извлеченный из последовательности `sst3` (максимально часто встречающийся символ C, H или E).
## 3. Используемые технологии и библиотеки

- **Pandas**, **NumPy** — обработка и анализ данных
- **Matplotlib**, **Seaborn** — визуализация результатов (графики, тепловые карты)

### Биоинформатика
- **BioPython** (ProteinAnalysis) — расчет физико-химических свойств белков

### Машинное обучение (Scikit-learn)
- **RandomForestClassifier**, **LogisticRegression**, **XGBoost** — модели классификации
- **train_test_split**, **StratifiedKFold** — разделение данных и кросс-валидация
- **GridSearchCV** — оптимизация гиперпараметров

- **Transformers** (Hugging Face) — загрузка предобученной модели ProtBERT

### Вспомогательные инструменты

## 4. Методология

### 4.1. Инжиниринг признаков
Для обучения моделей использовались три подхода к представлению последовательностей:
1.  **Физико-химические признаки:** Для каждой последовательности с помощью библиотеки `BioPython` были рассчитаны пять глобальных характеристик белка:
    *   Молекулярный вес
    *   Ароматичность
    *   Индекс нестабильности
    *   Гидрофобность (GRAVY)
    *   Изоэлектрическая точка

      ![Распределение таргетов](https://github.com/mazzeratziii/Predicting-the-secondary-structure-of-proteins/blob/main/images/%D1%80%D0%B0%D1%81%D0%BF%D1%80%D0%B5%D0%B4%D0%B5%D0%BB%D0%B5%D0%BD%D0%B8%D0%B5%20target.png)
2.  **K-меры (k=3):** Последовательности были представлены в виде векторов частот всех возможных трипептидов.
3.  **Эмбеддинги ProtBERT:** Использовалась предобученная трансформер-модель `ProtBERT` для получения плотных векторных представлений последовательностей.

### 4.2. Модели машинного обучения
Были обучены и сравнены следующие модели:
*   **Случайный лес (RandomForest):**
    *   На физико-химических признаках.
    *   На K-мерах.
    *   На эмбеддингах ProtBERT.
    *   **Тюнингованная версия:** Оптимизация гиперпараметров для модели на физико-химических признаках с помощью `GridSearchCV`.
*   **Логистическая регрессия (Logistic Regression):** Обучена на физико-химических признаках с балансировкой классов.
*   **XGBoost:** Обучен на физико-химических признаках с балансировкой классов.

### 4.3. Оценка моделей
Для оценки качества использовались метрики accuracy, матрицы ошибок, ROC-кривые и кривые обучения (learning curves).

Ниже представлено сравнение всех обученных моделей по метрике accuracy.

| Model | Accuracy |

## 6. Выводы
1.  **Лучшая модель:** Наиболее эффективным подходом в рамках данного исследования оказался **RandomForest с оптимизированными гиперпараметрами**, обученный на простых физико-химических признаках. Это говорит о том, что для данной задачи глобальные свойства белка являются очень информативными.
2.  **Сложные признаки:** Представления на основе K-меров и эмбеддингов ProtBERT не превзошли по качеству тюнингованный RandomForest на физико-химических признаках. Возможно, для раскрытия их потенциала требуется больше данных или более сложная модель (например, дообучение ProtBERT).
3.  **Линейные модели:** Логистическая регрессия показала низкий результат даже с балансировкой классов, что может указывать на сложную, нелинейную природу зависимости между признаками и целевой переменной.
4.  **Балансировка классов:** Использование балансировки классов в XGBoost и логистической регрессии было необходимо, но не привело к значительному улучшению качества по сравнению с лучшей моделью.
